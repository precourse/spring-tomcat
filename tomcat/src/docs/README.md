# 미션: Tomcat 구현하기

## 목적

HTTP/1.1 동작의 핵심을 직접 구현하고, 정적 자원 처리, 인증/세션, 라우팅 구조화, 동시성 처리를 단계적으로 확장한다.

## 공통 조건

- 포트는 8080에서 수신한다.
- netty, spring, tomcat 등 웹 프레임워크/서버 라이브러리를 사용하지 않는다.
- 요청을 받아 응답을 직접 생성한다.
- 정답은 브라우저에서 직접 동작 확인이 가능해야 하며, 테스트 코드로도 검증할 수 있어야 한다.
- 상태코드, 헤더, 본문은 HTTP 응답 규격을 맞춘다.

## 1단계

- GET /index.html 요청에 대해 200과 함께 정적 HTML을 반환한다.
- css, js 등 정적 자원을 조회해 반환할 수 있다.
- URI의 Query String(?key=value&...)을 파싱한다.

## 2단계

- POST 요청 바디를 읽어 파싱한다.
- /register로 회원가입을 처리한다.
- /login으로 로그인 처리를 수행한다.
- 로그인/회원가입 성공 시 302 Found로 리다이렉트한다.
- 로그인 성공 시 응답에 Set-Cookie: JSESSIONID를 포함한다.
- JSESSIONID를 기반으로 세션을 생성/관리한다.
- 로그인 실패 시 적절한 실패 응답(예: 401)을 반환한다.

## 3단계

- 요청을 HttpRequest, 응답을 HttpResponse로 분리해 모델링한다.
- Controller 인터페이스를 만들고 URL별 Controller로 요청을 위임한다.
- RequestMapping을 통해 경로 기반 라우팅을 한다.
- 지원하지 않는 경로는 404를 반환한다.

## 4단계

- 연결 수락/요청 처리를 ExecutorService 기반 스레드풀로 처리한다.
- 공유 상태(예: 세션, 사용자 저장소)는 동시성에 안전하게 관리한다.
